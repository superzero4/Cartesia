This method attempts to connect to a remote TCP server peer at a give IP/port. If the connection is successful, the socket is available for reading and writing data to/from the peer. The system transparently chooses any available local port to initiate the connection to the peer.

◆ accept()

cSocketTCP * chai3d::cSocketTCP::accept ( bool  a_block = true,
double  a_timeout = -1.0 
)

This method listens for incoming TCP connections and spawns a new socket for communication once the communication is established. Calls to accept must be preceded by a call to cSocketTCP::listen(). If cSocketTCP::listen() was passed the BLOCKING attribute, each call to accept() waits for an incoming connection. Otherwise, accept() returns immediately with WOULDBLOCK if no connection requests are pending.

Parameters
a_blockSet the BLOCKING attribute of the returned TCP socket (true by default).
a_timeoutSet an optional timeout value in seconds. A negative value means that the call will never timeout. This parameters is ignored if the socket is non-blocking.
Returns
A pointer to a valid cSocketTCP object connected to the peer making the connection request if successful, NULL otherwise.
